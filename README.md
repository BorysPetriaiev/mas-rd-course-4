# 🤖 AI Research Agent

Персональний автономний агент.

Агент працює за циклом **ReAct** (Reasoning + Acting): він обмірковує запит, планує дії, використовує інструменти та формує фінальний аналітичний звіт.



## 🛠 Технологічний стек
- **Python 3.12**
- **uv** 
- **OpenAI**

## 📦 Встановлення та запуск

### 1. Підготовка середовища
Переконайтеся, що у вас встановлено інструмент [uv](https://docs.astral.sh/uv/).

```bash
# Клонуйте проєкт та перейдіть у папку
cd mas-rd-course-4

# Синхронізуйте залежності
uv sync
```


### 2. Налаштування конфіденційних даних
Створіть файл .env у кореневій директорії та додайте ваші ключі:

```
OPENAI_API_KEY=your_openai_key_here
```

### 3. Запуск агента

```bash
uv run main.py
```

### Фактори, які впливають на вибір методу

1. **Тип даних**:
   - **Помічені дані**: Якщо у вас є значна кількість помічених даних, контрольоване навчання (такие як лінійна регресія, дерева рішень) буде найефективнішим.
   - **Немарковані дані**: Якщо дані не мають етикеток, неконтрольоване навчання (кластеризація, PCA) може бути корисним для виявлення структур.

2. **Ціль задачі**:
   - **Класифікація або регресія**: Для задач, що вимагають передбачення категорій, контрольоване навчання краще. Для прогнозування числових значень підходять регресійні моделі.
   - **Адаптивність**: Якщо система повинна навчатися з досвіду й приймати рішення на основі винагород, то навчання з підкріпленням є оптимальним.

3. **Складність задачі**:
   - **Складні патерни**: Для задач, пов'язаних з класифікацією зображень або обробкою мов, глибоке навчання з багаторівневими нейронними мережами показує найкращі результати.
   - **Простота**: Якщо задача проста, класичні методи (лінійна регресія, дерева рішень) можуть бути достатніми і менш ресурсомісткими.

4. **Обсяг даних**:
   - **Великий обсяг даних**: Глибоке навчання зазвичай вимагає великих обсягів даних для досягнення високої продуктивності.
   - **Маленький обсяг даних**: Класичні алгоритми можуть показати кращі результати при обмежених даних.
